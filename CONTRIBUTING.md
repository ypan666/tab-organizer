# Contributing to Tab Organizer

First, thank you for considering contributing to Tab Organizer!

## Code of Conduct

By participating in this project, you agree to maintain a respectful and constructive attitude.

## How to Contribute

### Reporting Bugs

If you find a bug, please create an issue with:

- **Clear Title** - Briefly describe the problem
- **Detailed Description** - Explain what the issue is
- **Steps to Reproduce** - How to replicate the problem
- **Expected Behavior** - What you expected to happen
- **Actual Behavior** - What actually happened
- **Environment Info** - Chrome version, operating system, etc.
- **Screenshots** - If applicable

### Suggesting Features

If you have a feature suggestion:

1. Check if a similar issue already exists
2. Create a new feature request
3. Describe the feature and use case in detail
4. Explain why this feature would be useful

### Submitting Code

1. Fork the repository
2. Create your feature branch: `git checkout -b feature/AmazingFeature`
3. Commit your changes: `git commit -m 'Add some AmazingFeature'`
4. Push to the branch: `git push origin feature/AmazingFeature`
5. Open a Pull Request

### Code Standards

- Use clear, meaningful variable and function names
- Add necessary comments (in English)
- Keep code style consistent
- Ensure all functionality is tested

### Commit Message Guidelines

Use clear commit messages:

```
feat: Add new feature
fix: Fix bug
docs: Update documentation
style: Code formatting
refactor: Code refactoring
test: Add tests
chore: Build or auxiliary tools changes
```

Examples:
```
feat: Add domain-based grouping feature
fix: Fix tab order issue during archiving
docs: Update installation instructions in README
```

## Development Setup

### Requirements
- Chrome browser (latest version)
- Text editor (VS Code recommended)
- Git

### Local Development

```bash
# Clone repository
git clone https://github.com/your-username/tab-organizer.git
cd tab-organizer

# Load extension in Chrome
# 1. Open chrome://extensions/
# 2. Enable developer mode
# 3. Click "Load unpacked"
# 4. Select project folder
```

### Testing

After modifying code:

1. Reload extension (click refresh on chrome://extensions/)
2. Test all related functionality
3. Check browser console for errors
4. Refer to test checklist for comprehensive testing
